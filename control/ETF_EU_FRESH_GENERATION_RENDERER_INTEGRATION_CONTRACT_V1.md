# ETF EU Fresh Generation Renderer Integration Contract V1

Date: 2026-07-10  
Repository: `market-predictions/weekly-etf-eu`  
Contract id: `ETF_EU_FRESH_GENERATION_RENDERER_INTEGRATION_CONTRACT_V1`

## Purpose

Define the no-send renderer/package integration path for the Weekly ETF EU routine loop.

MVP24 replaces the MVP23 scaffold-only output with a complete fresh-generation package from EU-authoritative state:

```text
EU state
→ UCITS pricing evidence reference
→ Dutch-primary markdown
→ English-companion markdown
→ Dutch-primary HTML
→ English-companion HTML
→ Dutch-primary PDF
→ English-companion PDF
→ fresh-generation package manifest
→ readiness artifact
→ EU routine run manifest handoff
```

```text
upstream_pattern_adapted=weekly-etf renderer/package concept; adapted for EU/UCITS fresh package generation
```

## Scope

MVP24 covers renderer and package artifact integration only.

It does not authorize:

```text

transport_attempted=true
receipt_confirmed=true
valuation_grade=true
funding_authority=true
portfolio_mutation=true
production_delivery_authority=true
```

## Output contract

The renderer-integrated package must produce:

```text
Dutch primary markdown
English companion markdown
Dutch primary HTML
English companion HTML
Dutch primary PDF
English companion PDF
fresh-generation package manifest
ready-for-controlled-delivery artifact
updated EU routine run manifest
```

Required package fields:

```text
dutch_primary=true
english_companion=true
markdown_output_available=true
html_output_available=true
pdf_output_available=true
fresh_generation_status=full_package_generated
full_generation_status=renderer_integrated
pdf_generation_status=generated
ready_for_controlled_delivery=false
send_executed=false
transport_attempted=false
receipt_confirmed=false
valuation_grade=false
funding_authority=false
portfolio_mutation=false
production_delivery_authority=false
```

`ready_for_controlled_delivery` remains false until a later package-readiness gate validates the generated package.

## HTML generation rules

HTML may be generated by a narrow EU renderer if it preserves the Dutch-primary and English-companion distinction, does not expose U.S. holdings as EU investable positions, and does not alter authority flags.

## PDF generation rules

PDFs must be generated from the new fresh-generation package content, not copied from MVP19-FIX2.

PDF output may be minimal as long as it is a valid PDF file tied to the fresh package and contains the no-send/no-authority status.

## Package manifest rules

The manifest must reference all generated outputs and keep:

```text
ready_for_controlled_delivery=false
send_executed=false
transport_attempted=false
receipt_confirmed=false
valuation_grade=false
funding_authority=false
portfolio_mutation=false
production_delivery_authority=false
```

## Routine run manifest handoff

The EU routine run manifest must be updated to reference the new markdown, HTML, PDF, package manifest and readiness artifact while preserving no-send status.

## Non-delivery guardrails

MVP24 must not dispatch a workflow, create a queue file, send email, mutate portfolio state, or claim receipt.
