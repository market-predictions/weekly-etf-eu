# ETF EU Fresh Package Readiness Gate Contract V1

Date: 2026-07-10  
Repository: `market-predictions/weekly-etf-eu`  
Contract id: `ETF_EU_FRESH_PACKAGE_READINESS_GATE_CONTRACT_V1`

## Purpose

Define the package-readiness gate for the fresh Weekly ETF EU package generated by MVP24.

This gate determines whether the generated Dutch-primary / English-companion package is eligible for a later controlled-delivery preparation package.

```text
upstream_pattern_adapted=weekly-etf package-readiness/pre-send validation concept; adapted for EU fresh package readiness without delivery authority
```

## Scope

MVP25 validates:

```text
Dutch-primary markdown
English-companion markdown
Dutch-primary HTML
English-companion HTML
Dutch-primary PDF
English-companion PDF
fresh-generation package manifest
ready-for-controlled-delivery artifact
EU routine run manifest handoff
```

MVP25 may set:

```text
ready_for_controlled_delivery=true
```

only after package gates pass.

MVP25 must not set:

```text
delivery_authorized=true
send_executed=true
transport_attempted=true
receipt_confirmed=true
valuation_grade=true
funding_authority=true
portfolio_mutation=true
production_delivery_authority=true
```

## Authority boundaries

```text
source_of_truth_repo=market-predictions/weekly-etf-eu
```

## Markdown gate

The markdown gate requires:

```text
Dutch markdown exists
English markdown exists
Dutch markdown title identifies Nederlands / Dutch primary
English markdown title identifies English Companion
both markdown files contain no-send / no-authority flags
no stale resend wording
no U.S. ETF symbols presented as investable EU holdings
```

## HTML gate

The HTML gate requires:

```text
Dutch HTML exists
English HTML exists
HTML contains expected title and body markers
HTML preserves Dutch-primary / English-companion distinction
HTML contains no stale resend wording
HTML contains no delivery-execution claim
```

## PDF gate

The PDF gate requires:

```text
Dutch PDF exists
English PDF exists
each PDF starts with %PDF-
each PDF contains an EOF marker
```

This is a structural gate only. Visual/premium-layout validation remains eligible for a later package if required.

## Manifest gate

The manifest gate requires:

```text
fresh_generation_status=full_package_generated
full_generation_status=renderer_integrated
markdown_output_available=true
html_output_available=true
pdf_output_available=true
dutch_primary=true
english_companion=true
send_executed=false
transport_attempted=false
receipt_confirmed=false
valuation_grade=false
funding_authority=false
portfolio_mutation=false
production_delivery_authority=false
EU-specific state paths
all referenced package files exist
```

## Authority gate

The authority gate requires:

```text
delivery_authorized=false
send_executed=false
transport_attempted=false
receipt_confirmed=false
valuation_grade=false
funding_authority=false
portfolio_mutation=false
production_delivery_authority=false
```

`ready_for_controlled_delivery=true` is not delivery authorization. It only means the package is eligible for a later explicit delivery-prep package.

## Readiness decision rules

If all gates pass:

```text
readiness_gate_passed=true
ready_for_controlled_delivery=true
blockers=[]
next_package=ETF-EU-MVP26_GUARDED_FRESH_PACKAGE_DELIVERY_PREP
```

If any gate fails:

```text
readiness_gate_passed=false
ready_for_controlled_delivery=false
blockers=<explicit blockers>
next_package=ETF-EU-MVP25B_FRESH_PACKAGE_READINESS_FIXES
```

with:


The routine run manifest must record:

```text

workflow_status=fresh_package_readiness_gate_passed
package_readiness_gate=<gate artifact path>
ready_for_controlled_delivery=true

transport_success=false
receipt_confirmed=false
next_package=ETF-EU-MVP26_GUARDED_FRESH_PACKAGE_DELIVERY_PREP
```

## Non-delivery guardrails

MVP25 must not:

```text
send email
dispatch workflow
create run queue file
perform live delivery
mutate portfolio state
create funded UCITS positions
confirm receipt for a new run
store raw Gmail PDF in GitHub
copy U.S. ETF portfolio state
use U.S. ETF tickers as investable EU holdings
treat ready_for_controlled_delivery=true as send authorization
merge fresh generation and delivery into one production workflow
```
